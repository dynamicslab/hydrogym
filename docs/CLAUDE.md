# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Overview

This is the **documentation site** for HydroGym, a reinforcement learning platform for fluid dynamics. The docs are built with Docusaurus and live in the `/docs` subdirectory of the main HydroGym repository.

HydroGym provides 88 RL environments across 6 CFD solver backends (Firedrake, MAIA LBM, MAIA FV, NEK5000, JAX, JAX-Fluids) for training agents on flow control problems.

## Common Commands

### Documentation (this directory)

```bash
# Install dependencies
yarn

# Start local development server (with hot reload)
yarn start

# Build static site
yarn build

# Type checking
yarn typecheck
```

### Python Package (parent directory)

```bash
# Install in development mode
pip install -e .

# Install with solver-specific extras
pip install -e ".[firedrake]"
pip install -e ".[maia]"
pip install -e ".[nek]"
pip install -e ".[all]"

# Linting and formatting
ruff check .
ruff format .
isort .

# Spell check
codespell

# Run tests (requires Firedrake Docker container)
cd test && python -m pytest .
python -m pytest test_pinball.py  # single test file
```

## Architecture

### Documentation Structure (Docusaurus)

- `docs/` - Markdown documentation pages (auto-generates sidebar)
- `src/` - React components and custom CSS
- `static/` - Static assets (images, favicon)
- `docusaurus.config.ts` - Site configuration (URL, navbar, footer)
- `sidebars.ts` - Sidebar configuration (uses autogenerated from filesystem)

### Python Package Structure (../hydrogym/)

- `core.py` - Base classes: `FlowEnv`, `PDEBase`, `TransientSolver`, `CallbackBase`
- `data_manager.py` - Hugging Face Hub integration for checkpoints/configs
- `firedrake/` - Firedrake FEM solver environments (2D canonical benchmarks)
- `maia/` - MAIA LBM and structured FV solver environments
- `nek/` - NEK5000 spectral element solver environments
- `jax/` - Differentiable JAX-based solvers
- `distributed/` - Multi-agent and distributed RL support

### Key Design Patterns

- Lazy imports in `__init__.py` to avoid MPI initialization conflicts in MPMD mode
- Solver backends are optional extras - users install only what they need
- Environments follow Gymnasium API for compatibility with SB3, RLlib, etc.
- MPMD execution pattern for solvers (Python RL process + solver processes via MPI)

## CI/CD

GitHub Actions workflows:
- `build.yml` - Package build test with Poetry
- `ruff_lint.yml`, `ruff_format.yml`, `isort.yml` - Code formatting checks
- `codespell.yml` - Spell checking
- `deploy.yml` - Docs deployment to GitHub Pages

## Testing

Tests require the Firedrake Docker container (`lpaehler/hydrogym-env:stable`) due to complex solver dependencies. Run tests inside the container after activating the Firedrake virtualenv.
